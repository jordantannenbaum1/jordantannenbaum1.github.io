01 — The Story

Eighty years. Twenty-four orders of magnitude. No precedent.

When ENIAC came online in 1945, it performed roughly 5,000 operations per second. It weighed 30 tons, occupied 1,800 square feet, and consumed 150 kilowatts of power. At the time, it was the most powerful calculating machine humanity had ever built.

By 2025, training a single AI model — Grok-3 — consumed an estimated 10²⁷ floating-point operations. That's a number so large it defies intuition. It's more than the number of grains of sand on every beach on Earth, multiplied by a billion.

"The compute growth since 1945 is larger than the gap between a single neuron and every human brain that has ever existed — combined."

This isn't a metaphor. It's a mathematical fact, and the charts on this page were built to make that viscerally clear. The data is sourced from Epoch AI, TOP500.org, the xAI Colossus documentation, and Oak Ridge National Laboratory. Every number is cited. Every transformation is shown.

Why this matters

The standard narrative around AI focuses on capabilities: what models can do, what jobs they might replace, whether they're "intelligent." But the compute story is more fundamental. We have never scaled anything this fast. Not steel production during WWII. Not transistor counts. Not internet bandwidth. Nothing.

The Big Bang expanded the observable universe by roughly 41 orders of magnitude over 13.8 billion years. Human computation expanded 24 orders of magnitude in 80 years — and is still accelerating. At current rates, computation will match the full scale of cosmic expansion within centuries, not billions of years.

Methodological note: FLOPs are not a measure of "intelligence." They measure raw compute consumed. Comparisons between neuron counts and FLOPs are analogical, not equivalent — the charts in section 4 are explicitly labeled as scale comparisons. Projected values carry ±1 order of magnitude uncertainty.

03 — The Pipeline

From raw CSVs to a story about cosmic scale.

The full five-stage pipeline — ingest, clean, transform, QA, visualize. Each stage is expandable with the actual code.

1
Ingest — pulling raw data
Python ▾

Two automated sources (Epoch AI, TOP500) pulled via direct download. Manual data points loaded from a hand-maintained seed CSV.

ingest/fetch_sources.pyPython 3.11
import pandas as pd
import requests
from pathlib import Path
from datetime import datetime

RAW_DIR = Path("data/raw")
RAW_DIR.mkdir(parents=True, exist_ok=True)

EPOCH_AI_URL = "https://epochai.org/data/notable-ai-models.csv"

def fetch_epoch_ai():
    response = requests.get(EPOCH_AI_URL, timeout=30)
    response.raise_for_status()
    path = RAW_DIR / f"epoch_ai_{datetime.today().strftime('%Y%m%d')}.csv"
    path.write_bytes(response.content)
    return pd.read_csv(path)

def load_top500(filepath: str) -> pd.DataFrame:
    # Manual download from top500.org/statistics/
    df = pd.read_csv(filepath)
    return df[["Name", "Rmax [TFlop/s]", "Year", "Manufacturer"]]

def load_seed_data() -> pd.DataFrame:
    # ENIAC, neuron counts, cosmology constants
    return pd.read_csv("data/seed/manual_datapoints.csv")
2
Clean — standardizing units and nulls
Python ▾

Core challenge: Epoch AI reports FLOPs (total training compute) while TOP500 reports GFlop/s (peak throughput). Both normalized to log₁₀ scale with different series tags to preserve the distinction.

clean/normalize_flops.pyPython 3.11
import numpy as np

UNIT_MAP = {
    "FLOP/s": 1, "KFLOP/s": 1e3, "MFLOP/s": 1e6,
    "GFLOP/s": 1e9, "TFLOP/s": 1e12,
    "PFLOP/s": 1e15, "EFLOP/s": 1e18,
}

def clean_epoch_ai(df):
    df = df.dropna(subset=["Training compute (FLOP)"])
    df = df[df["Training compute (FLOP)"] > 0]
    df["log10_flops"] = np.log10(df["Training compute (FLOP)"])
    df["year"] = pd.to_datetime(df["Publication date"]).dt.year
    df["series"] = "ai_training"
    return df[["Name", "year", "log10_flops", "series"]]

def clean_top500(df):
    # Rmax is in TFlop/s; take #1 system per year
    df["log10_flops"] = np.log10(df["Rmax [TFlop/s]"] * 1e12)
    df["series"] = "supercomputer"
    return df.groupby("year").apply(
        lambda x: x.nlargest(1, "log10_flops")
    ).reset_index(drop=True)
Key decision: Total training FLOPs (Epoch AI) vs. peak throughput (TOP500) are not directly comparable — one is a total, one is a rate. Both appear on the same log scale as proxies for "how much compute exists," with different visual encodings to signal this distinction.
3
Transform — dbt models
dbt SQL ▾

Three dbt models produce the analytical tables. Staging normalizes raw sources. The mart adds order-of-magnitude jump calculations and milestone tagging. A separate model unifies the biological + silicon comparison.

models/mart/mart_compute_timeline.sqldbt / Snowflake SQL
with events as (
    select * from {{ ref('stg_compute_events') }}
),
milestones as (
    select
        *,
        log10_flops - lag(log10_flops) over (
            partition by series order by year
        ) as orders_of_magnitude_jump
    from events
)
select
    *,
    case
        when orders_of_magnitude_jump >= 3 then 'major_milestone'
        when orders_of_magnitude_jump >= 1 then 'milestone'
        else 'incremental'
    end as milestone_type
from milestones
order by year, series
models/mart/mart_bio_vs_silicon.sqldbt / Snowflake SQL
-- Unified scale table: biological neuron counts + silicon FLOPs
-- Note: neurons ≠ FLOPs. Scale comparison only.
select
    entity_name,
    'biological' as domain,
    log10_neuron_count as log10_scale_value,
    description
from {{ ref('stg_biological_entities') }}

union all

select
    system_name,
    'silicon',
    log10_flops,
    notes
from {{ ref('stg_compute_events') }}
where system_name in (
    'ENIAC','Cray-1','ASCI Red','IBM Roadrunner',
    'IBM Summit','Frontier','GPT-3','GPT-4','Gemini Ultra','Grok-3'
)
4
QA — dbt tests and sanity checks
dbt ▾

Schema tests enforce nulls, uniqueness, and valid ranges. A pytest suite spot-checks known values against primary sources with ±0.1 log₁₀ tolerance.

models/mart/mart_compute_timeline.ymldbt schema tests
models:
  - name: mart_compute_timeline
    columns:
      - name: system_name
        tests: [not_null, unique]
      - name: year
        tests:
          - dbt_utils.accepted_range:
              min_value: 1940
              max_value: 2027
      - name: log10_flops
        tests:
          - dbt_utils.accepted_range:
              min_value: 3   # ENIAC floor
              max_value: 30  # conservative 2026 ceiling
      - name: series
        tests:
          - accepted_values:
              values: ['supercomputer', 'ai_training']
tests/test_known_values.pypytest
KNOWN_VALUES = {
    "ENIAC":          {"log10": 3.7,  "source": "Wikipedia"},
    "ASCI Red":       {"log10": 12.0, "source": "TOP500"},
    "IBM Roadrunner": {"log10": 15.0, "source": "TOP500"},
    "Frontier":       {"log10": 18.08,"source": "ORNL"},
    "GPT-3":          {"log10": 23.49,"source": "Epoch AI"},
}

def test_known_values(mart_df):
    for name, expected in KNOWN_VALUES.items():
        row = mart_df[mart_df["system_name"] == name]
        actual = row["log10_flops"].values[0]
        # ±0.1 log10 tolerance — order-of-magnitude estimates
        assert abs(actual - expected["log10"]) < 0.1

Visualize — D3.js chart methodology
D3.js v7 ▾

All four charts are built with D3.js v7, sharing a common dark theme and log-scale formatting. Key design decisions:

Why log scale? A linear scale is useless — ENIAC would be invisible at any resolution that shows Grok-3. The log scale is not a trick; it's the only honest representation of 24 orders of magnitude.

Why are neurons and FLOPs on the same axis in Chart 3? They measure different things. The chart forces the same axis to illustrate a point about scale, not equivalence. Every labeled entity carries a description of what the number actually means, and the subtitle makes the analogical nature explicit.

Why include projected values? The 2026 Frontier projection represents publicly announced infrastructure. Excluding it truncates the story at the most interesting edge. Projected values are visually differentiated (reduced opacity) and documented in the source table.

assets/charts.js — shared configD3.js v7
// Shared palette — all charts use these constants
const C = {
  cyan:   "#00d4ff",  // supercomputers / HPC
  pink:   "#ff4d9d",  // AI training runs
  green:  "#06d6a0",  // cosmos / silicon (Chart 4)
  yellow: "#ffd166",  // annotations / gap markers
  muted:  "#8892a4",  // labels
};

// Log-scale Y axis formatter
// Maps raw values to human-readable SI prefixes + scientific notation
function logLabel(v) {
  const exp = Math.round(Math.log10(v));
  const labels = {
    3:"1K", 6:"1M", 9:"1G", 12:"1T",
    15:"1P", 18:"1E", 21:"1Z", 24:"1Y",
    27:"10²⁷", 30:"10³⁰"
  };
  return labels[exp] || `10^${exp}`;
}
04 — The Visualizations

Four ways to look at a number that doesn't fit in your head.

Each chart makes the same underlying point from a different angle. Read them in order — they build on each other.

Chart 1 — Maximum computational power, 1945–2026 log scale · FLOP/s
Sources: Epoch AI, TOP500.org, xAI Colossus, ORNL · AI training FLOPs are published/estimated values · Projected values shown with dashed line

The two series (supercomputer throughput in cyan, AI training in pink) diverge sharply after 2020. Frontier — the world's most powerful supercomputer — sits at 10¹⁸ FLOPs. Grok-3's training run consumed 10²⁷ FLOPs. That's 9 orders of magnitude beyond the fastest supercomputer — a billion-fold gap, opened in roughly five years.

Chart 2 — Biological intelligence vs. machine computation log scale · unified axis
Biological neuron counts and silicon training FLOPs on the same log₁₀ axis. Scale comparison only — neurons ≠ FLOPs. Everything biological sits below 10¹⁵. Sources: Azevedo et al. 2009, Herculano-Houzel 2009, Epoch AI, xAI.

On a single axis, the entire range of biological intelligence — from one neuron to every neuron in every human who ever lived (~10¹⁵) — occupies the bottom third of the chart. The top two-thirds is a region where nothing biological has ever existed. Grok-3's training run sits there. That's not editorial. It's a description of empty space.

Chart 3 — The Big Bang vs. the compute explosion log scale · cosmological
Left: expansion of the observable universe, Planck length to present (~41 orders in 13.8B years). Right: computation from ENIAC to projected 2026 (~24 orders in 80 years). Sources: NIST, NASA cosmology constants, Epoch AI.

The Big Bang expanded by ~41 orders of magnitude over 13.8 billion years. Human computation expanded ~24 orders in 80 years — and is still accelerating. Both series plotted on the same log scale. At current doubling rates, computation will match the full scale of cosmic expansion within centuries, not billions of years.

Chart 4 — Atoms in the observable universe vs. human computation log scale · 10⁷⁸ ceiling
Left: observable universe scale milestones, anchored to ~10⁷⁸ atoms today (Eddington number). Right: human computation, ENIAC to 2026. Sources: standard cosmology, Epoch AI, TOP500.org, ORNL, xAI. Note: intermediate scale values are epoch proxies, not literal atom counts.

The observable universe contains approximately 10⁷⁸ atoms — a number it took 13.8 billion years and every star that ever existed to produce. Human computation started at 10³·⁷ in 1946 and reached 10²⁷ by 2025: 24 orders of magnitude in 80 years. At the current doubling rate (~12 months), computation could arithmetically reach 10⁷⁸ operations within a single human lifetime. Whether that number means anything at the scale of atoms is not the point. The point is that we are scaling faster than anything in the history of physics.

05 — Key Findings

What the data actually shows.

24x

Orders of magnitude in 80 years — larger than the gap between one neuron and all biology ever combined

9 OOM

Gap between the world's fastest supercomputer (Frontier, 10¹⁸) and Grok-3 training (10²⁷) — opened in ~5 years

10¹⁵

Ceiling of all biological computation ever. Every neuron, every brain, every organism. Machine compute passed this ~2020.

~2×/yr

Rough doubling rate of frontier AI training compute since 2010, faster than Moore's Law and still accelerating

The question isn't whether this growth is "good" or "bad." The question is whether we have frameworks adequate to reason about what we're building — and whether we're building them fast enough.

What this project doesn't answer

Compute is a proxy, not a measure of intelligence. Efficiency gains mean later models do more per FLOP. The biological comparisons are analogical. This project makes no claim about AI consciousness or capability trajectories — only about the scale of infrastructure we've built in 80 years.
